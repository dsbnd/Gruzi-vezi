**Грузи Вези** — это платформа для автоматизации грузовых железнодорожных перевозок. Система решает проблему хаотичного поиска подвижного состава и сложности расчета стоимости перевозок, предоставляя единую среду для взаимодействия грузоотправителей и перевозчика.

Организация железнодорожной перевозки часто сопряжена с разрозненным поиском вагонов, сложностями расчета тарифов и потерей информации о заказах.

Наш сервис устраняет эти проблемы:

- **Централизация данных:** Платформа служит единым источником информации о заявках, вагонах и статусах перевозок.
- **Упрощение поиска:** Умный алгоритм подбирает оптимальный подвижной состав с учетом характеристик груза и доступности.
- **Прозрачные расчеты:** Автоматический расчет стоимости с учетом типа груза, расстояния и выбранных дополнительных услуг.

## Ключевой функционал

### 1. Интеллектуальный поиск и резервирование вагонов

Реализован модуль поиска подвижного состава с учетом множества параметров. Алгоритм анализирует грузоподъемность, объем, текущее местоположение вагона и его занятость в расписании. При нехватке вагонов на выбранной станции система предлагает альтернативные варианты с ближайших станций, рассчитывая время подачи.

- **Валидация совместимости:** Система проверяет соответствие типа груза типу вагона (наливные грузы только в цистернах, контейнеры только на платформах) и плотность груза.
- **Резервирование:** Временная блокировка выбранного вагона с помощью Redis на 30 минут.

### 2. Расчет стоимости перевозки

- **Тарифная сетка:** Более 20 тарифов для различных комбинаций "тип вагона — тип груза".
- **Расстояние:** Автоматическое определение километража между станциями.
- **Дополнительные услуги:** Динамический расчет стоимости страхования, сопровождения, терминальной обработки и ускоренной доставки.

### 3. Корпоративные платежи и документооборот

- **Банковские счета:** Управление несколькими расчетными счетами для одной компании.
- **Проведение платежей:** Перевод средств со счета грузоотправителя на счет перевозчика с атомарной блокировкой транзакций.
- **Генерация PDF:** Автоматическое формирование счетов на оплату и договоров перевозки с подписями.

### 4. Безопасность и управление доступом

- **JWT аутентификация:** Access и refresh токены с хранением refresh токенов в Redis.
- **Ролевая модель:** Разделение прав для пользователей и администраторов.

## Архитектура и Технологический стек

Проект построен по принципам многоуровневой архитектуры, где клиент и сервер общаются через RESTful API.

- **Backend:** Java 17, Spring Boot 3.5.7, Spring Security, Spring Data JPA.
- **ORM:** Hibernate 6.2.0.
- **База данных:** PostgreSQL 17 (основная), Redis 7 (кэширование, резервирование, токены).
- **Документооборот:** OpenPDF для генерации договоров и счетов.
- **Сборка:** Gradle 8.5.

## Проектирование Базы Данных

БД спроектирована с упором на надежность, целостность данных и высокую производительность.

- Включает более 15 сущностей с отношениями "один-ко-многим" и "многие-ко-многим".
- Обеспечена строгая ссылочная целостность средствами DDL с каскадным удалением и ограничениями уникальности.
- Реализованы триггеры для автоматического создания банковского счета при регистрации нового пользователя.
- Внедрены B-Tree индексы для оптимизации поиска доступных вагонов по станции, статусу и характеристикам.
- Используются ENUM типы для ограничения допустимых значений статусов и типов.
- Реализована блокировка на уровне БД (PESSIMISTIC_WRITE) для предотвращения конкурентных списаний со счетов.

## Установка

```bash
# Клонирование репозитория
git clone https://github.com/dsbnd/Gruzi-vezi

# Создать базу данных
createdb -U postgres gruzi_vezi

# Восстановить из дампа (файл gruzi_vezi.sql)
psql -U postgres -d gruzi_vezi < gruzi_vezi.sql

# Скачать Redis и запустить как сервис для вашей ОС
brew services start redis          # macOS
sudo systemctl start redis         # Ubuntu/Debian

# Запуск приложения
./gradlew bootRun
